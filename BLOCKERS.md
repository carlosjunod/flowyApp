# Blockers & environment notes

Items logged here during autonomous execution. Code was written anyway; these require a macOS+Xcode machine to actually ship.

## Environment (Linux sandbox)

- **No Xcode / macOS** — `npx expo prebuild --platforms ios` cannot run here. The `ios/` project and the Share Extension Xcode target are generated on first prebuild. The `plugins/withShareExtension.ts` plugin + `ShareViewController.swift` source are committed and will be applied on prebuild.
- **Expo API host unreachable** from this sandbox (`api.expo.dev` returned "Host not in CORS"). This only affected `expo install` version-resolution; packages were installed directly via npm pinned to Expo SDK 54–compatible versions.
- **Cannot run `expo run:ios`** — requires CocoaPods + Xcode. Done-criteria item "launches and login works" cannot be manually verified here.

## What to do on a Mac after cloning

```bash
npm install
cp .env.example .env          # point at real backend hosts
npx expo prebuild --platforms ios
cd ios && pod install && cd ..
npx expo run:ios
```

For macOS (Mac Catalyst): in Xcode, target → Supported Destinations → add "Mac (Mac Catalyst)". The share extension target is auto-generated by the `withShareExtension` plugin.

## Decisions taken autonomously (final solutions, no workarounds)

- **PB auth persistence**: custom `SecureStoreAuthStore` wraps `expo-secure-store` with `keychainAccessGroup: 'group.app.tryflowy'`. Shared Keychain item key: `pb_auth` (stores `{token, model}` JSON). Same key is read by the Share Extension.
- **Streaming chat**: native `fetch` + `response.body.getReader()`. Hermes + RN 0.81 supports this. No polyfill shim committed.
- **Responsive grid columns**: `useWindowDimensions` — width < 640 → 2 cols, 640–1024 → 3, ≥1024 → 4.
- **Subscription vs polling**: start PocketBase realtime `subscribe` immediately after ingest; also tick a 3 s `getOne` poll while status is `pending`/`processing`. Unsubscribe + stop polling on `ready`/`error`.
- **Bundle identifiers**: main app `app.tryflowy.client`, share extension `app.tryflowy.client.ShareExtension`. App Group `group.app.tryflowy`.
- **Youtube/TikTok/Instagram host classification** in share extension: hostname suffix match against `youtube.com`, `youtu.be`, `tiktok.com`, `instagram.com`.
- **Share extension networking**: uses `URLSession` default config; sets `Authorization: Bearer <token>` header from shared keychain. Base URL baked in at build time via `Info.plist` `API_BASE_URL` entry injected by the config plugin from `process.env.EXPO_PUBLIC_API_BASE_URL`.
